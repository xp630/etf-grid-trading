## 1. 回测正确性验证

### 问题发现：浮点数精度导致基准价触发买入

**问题描述**：
原代码中 `current_price <= level_price` 的判断方式，在浮点数精度问题下会在基准价错误触发买入。

**修复方案**：
```python
# 修复前
if current_price < level_price and i < len(grid_levels) - 1:

# 修复后 - 使用 baseline_index 限制买入范围
baseline_index = len(grid_levels) // 2
if i < baseline_index and current_price < level_price:
```

**验证结果**：
- 简单波动测试通过：买→卖 正确执行
- 盈利计算正确：买入价×数量×(1+佣金+滑点) vs 卖出价×数量×(1-佣金-滑点)

### 3.1 趋势过滤网格策略

- `BUY_ONLY`：下跌趋势，只买不卖

**测试结果**：

## 4. 自动计算单位测试

### 核心问题
交易单位和投入资金强相关，需要根据本金自动计算。

### 自动计算公式

```python
每格单位 = (总资金 × 持仓比例) ÷ (买入档位数 × 当前价格)

示例（本金10,000元）：
  = (10,000 × 0.50) ÷ (5 × 4.0)
  = 5,000 ÷ 20
  = 250股 → 取整200股
```

### 测试结果（保守配置 50% 持仓）

| 本金 | 自动单位 | 收益率 | 年化 | 最大回撤 |
|------|----------|--------|------|----------|
| 10,000元 | 200股 | +0.69% | 0.70% | 0.20% |
| 20,000元 | 500股 | +0.87% | 0.87% | 0.25% |
| 30,000元 | 700股 | +0.81% | 0.81% | 0.23% |
| 50,000元 | 1200股 | +0.83% | 0.84% | 0.24% |
| 100,000元 | 2500股 | +0.87% | 0.87% | 0.25% |

**结论**：自动计算在不同本金下收益率稳定在 ~0.8%，回撤极低（<0.3%），适合追求稳健的用户。

## 5. 最终推荐配置

### 保守配置（推荐）

```yaml

  levels: 10           # 10档
  spacing: 0.05       # 5%间距

  enabled: true
  ma_period: 20        # MA20判断趋势
  trend_threshold: 0.05  # ±5%阈值

  enabled: true
  position_ratio: 0.50  # 最多50%资金持仓

  stop_loss_pct: 0.03  # 止损3%
```
