# 网格交易详解

> 目标用户：了解系统后配置参数、调整策略的用户和开发者。

## 1. 什么是"网格"

把价格区间想象成一个个"格子"，每个格子代表一个价格档位：

```
价格
  ↑
  │  ─ ─ ─ 第10档 4.797 (卖出线)
  │
  │  ─ ─ ─ 第9档  4.747
  │
  │  ─ ─ ─ 第8档  4.696
  │
  │  ─ ─ ─ 第7档  4.646
  │
  │  ─ ─ ─ 第6档  4.595
  │
  │  ─ ─ ─ 第5档  4.545
  │
  │  ─ ─ ─ 第4档  4.494
  │
  │  ─ ─ ─ 第3档  4.444
  │
  │  ─ ─ ─ 第2档  4.393
  │
  │  ─ ─ ─ 第1档  4.343 ← 买入线
  │
  │  ─ ─ ─ 基准价 4.569 (前一交易日收盘价)
  │
  └──────────────────────────────────────────────▶ 时间
```

### 1.1 网格参数计算

**当前配置**（`config.yaml`）：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| levels | 10 | 网格格数（单边） |
| spacing | 0.05 | 网格间距（5%） |
| base_price | 4.569 | 基准价（收盘价，自动获取） |

**计算公式**：

```
每格价格差 = 基准价 × 间距 = 4.569 × 0.05 = 0.228 元

第N档买入价 = 基准价 - (N × 每格价格差)
第N档卖出价 = 基准价 + (N × 每格价格差)
```

**实际档位价格表**（levels=10, spacing=5%）：

| 档位 | 类型 | 价格 | 说明 |
|------|------|------|------|
| 第1档 | 买入 | 4.343 | 价格跌到这里，买入 |
| 第2档 | 买入 | 4.393 | |
| 第3档 | 买入 | 4.444 | |
| 第4档 | 买入 | 4.494 | |
| 第5档 | 基准 | 4.545 | 上下分界线 |
| 第6档 | 卖出 | 4.595 | |
| 第7档 | 卖出 | 4.646 | |
| 第8档 | 卖出 | 4.696 | |
| 第9档 | 卖出 | 4.747 | |
| 第10档 | 卖出 | 4.797 | 价格涨到这里，卖出 |

### 1.2 网格交易赚钱原理

```
举例（简化版，每格买卖100股）：

第1步：价格从4.569跌到4.343
        → 触发第1档买入信号
        → 买入100股，花费 4.343 × 100 = 434.3元

第2步：价格反弹到4.545（中线）
        → 不买不卖，等待

第3步：价格继续涨到4.797（第10档）
        → 触发第10档卖出信号
        → 卖出之前买入的100股，收入 4.797 × 100 = 479.7元

盈利 = 479.7 - 434.3 = 45.4元

即使价格回到原点（4.569），我们也赚了45.4元！
```

**核心思想**：不管价格涨跌，只要波动就有收益。

## 2. 增强版：TrendGridStrategy

> 基础版 `GridStrategy` 适合入门，增强版 `TrendGridStrategy` 是生产环境推荐使用的策略。

### 2.1 趋势过滤机制

通过 MA20 均线判断市场趋势，三种模式自动切换：

| 模式 | 条件 | 行为 |
|------|------|------|
| `MODE_FULL_GRID` | 震荡市 | 正常网格，低买高卖 |
| `MODE_BUY_ONLY` | 下跌趋势 | 只买不卖，等待反弹 |
| `MODE_SELL_ONLY` | 上涨趋势 | 只卖不买，等待回调 |

**判断公式**：
```
上涨趋势：price > MA20 × (1 + threshold)   → SELL_ONLY
下跌趋势：price < MA20 × (1 - threshold)   → BUY_ONLY
震荡市：  其他情况                          → FULL_GRID
```

**阈值配置**（`config.yaml`）：
```yaml
trend_filter:
  enabled: true
  ma_period: 20        # MA 周期
  trend_threshold: 0.05  # ±5% 作为趋势判断门限
  confirm_days: 1      # 确认天数（连续N天确认趋势）
```

### 2.2 AI 网格间距调整

根据 AI 市场分析自动调整网格间距：

```yaml
# config.yaml 中的 ai_model.provider 配置
# 牛市：spacing → 6%
# 熊市：spacing → 4%
# 震荡：spacing → 5%（默认）
# 高波动：spacing → 3%
```

AI 分析依据：MA5/MA20/MA60 多空排列、波动率、价格动量。

### 2.3 自动单位计算

手动设置 `unit_size` 容易导致资金利用率不稳定。启用自动计算：

```yaml
auto_unit:
  enabled: true
  position_ratio: 0.5   # 最多50%资金用于持仓
  min_unit: 100        # 最小100股
```

**公式**：
```
每格单位 = (总资金 × 持仓比例) ÷ (买入档位数 × 当前价格)
示例（本金10000元，价格4.0元）：
  = (10000 × 0.5) ÷ (5 × 4.0) = 250股 → 取整200股
```

### 2.4 止损止盈机制

| 机制 | 默认值 | 说明 |
|------|--------|------|
| 止损 | 3% | 单笔亏损达到3%自动止损 |
| 止盈 | 8% | 单笔盈利达到8%自动止盈 |
| 移动止损 | 2% | 从最高点回落2%强制卖出 |

**优先级**：止损 > 止盈 > 移动止损 > 网格卖出

## 3. 信号生成详解

### 3.1 基础版 GridStrategy 信号逻辑

**买入信号**：
```
price < baseline（价格低于基准价）
    AND price < level_price（跌破了某档位）
    AND level empty（该档位无持仓）
    → 买入
```

**卖出信号**：
```
price > level_price（价格涨过了某档位）
    AND level has position（该档位有持仓）
    → 卖出
```

**约束**：每次最多处理 1 个信号，优先处理最靠近当前价格的档位。

### 3.2 增强版 TrendGridStrategy 信号逻辑

**买入**（`MODE_BUY_ONLY` 或 `MODE_FULL_GRID`）：
```
1. 检查模式是否允许买入
2. 计算自动买入数量（auto_unit 或固定 unit_size）
3. 检查止损条件（亏损 < 3%）
4. 检查是否已达止盈/移动止损
5. 检查档位是否空仓
6. 生成买入信号
```

**卖出**（`MODE_SELL_ONLY` 或 `MODE_FULL_GRID`）：
```
1. 检查模式是否允许卖出
2. 检查止损（亏损 ≥ 3%）
3. 检查止盈（盈利 ≥ 8%）
4. 检查移动止损（从最高点回落 ≥ 2%）
5. 检查档位是否有持仓
6. 生成卖出信号
```

## 4. 配置参数说明

### 4.1 完整配置（config.yaml）

```yaml
grid:
  levels: 10           # 网格格数（单边）
  spacing: 0.05       # 网格间距（5%）
  unit_size: 500       # 每格交易金额（元），仅 auto_unit=disabled 时生效

risk:
  initial_capital: 10000  # 初始本金（元）
  max_position: 5000   # 最大持仓（元）
  max_daily_loss: 100  # 日最大亏损（元）
  total_stop_loss: 9000  # 总资产止损线（元）

market:
  etf_code: 510300     # 沪深300ETF
  trading_hours:
    open: "09:30"
    close: "15:00"
    cancel_before: "14:50"

data_source:
  index: akshare       # 默认 AkShare（免费）
  # index: joinquant   # 聚宽（需要 JQCLOUD_USERNAME/JQCLOUD_PASSWORD）
  # index: tushare     # Tushare（需要 TUSHARE_TOKEN）

ai_model:
  provider: minimax   # MiniMax / openai / deepseek
  model: MiniMax-M2.7-highspeed
  api_key: xxx        # 环境变量：AI_API_KEY

trend_filter:
  enabled: true
  ma_period: 20
  trend_threshold: 0.05
  confirm_days: 1

auto_unit:
  enabled: true
  position_ratio: 0.5
  min_unit: 100

risk_control:
  stop_loss_pct: 0.03    # 3% 止损
  take_profit_pct: 0.08  # 8% 止盈
  trailing_stop_pct: 0.02  # 2% 移动止损

notification:
  server酱_key: xxx    # 环境变量：SERVERCHAN_KEY
```

### 4.2 参数调整建议

| 参数 | 默认值 | 调整场景 |
|------|--------|---------|
| levels | 10 | 资金少→减少格数，资金多→增加格数 |
| spacing | 0.05 | 波动大→增大间距，波动小→减小间距 |
| unit_size | 500 | 根据总资金调整，建议总资金的5-10% |
| max_position | 5000 | 不超过总资金的50% |
| max_daily_loss | 100 | 保守可设更低 |
| stop_loss_pct | 0.03 | 保守可设2%，激进可设4% |
| take_profit_pct | 0.08 | 宽幅震荡可设更高 |
| trailing_stop_pct | 0.02 | 趋势明显时效果最好 |

## 5. 风控机制

### 5.1 四重风控保护

```
┌─────────────────────────────────────────────────────────────┐
│                      风控体系                                │
├─────────────────────────────────────────────────────────────┤
│  第1重：买入前仓位检查                                        │
│  └─ 验证 new_position ≤ 5000 元                              │
│                                                             │
│  第2重：日亏损熔断                                            │
│  └─ 当日亏损 ≥ 100 元，停止交易，等待下一日                    │
│                                                             │
│  第3重：总资产止损                                            │
│  └─ 总资产 ≤ 9000 元，永久停止交易（需重启系统）               │
│                                                             │
│  第4重：止损止盈                                              │
│  └─ 单笔 3% 止损 / 8% 止盈 / 2% 移动止损                     │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 风控触发行为

| 触发条件 | 系统行为 |
|---------|---------|
| 仓位超限 | 拒绝下单，记录原因 |
| 日亏损达限 | 发送微信通知，等待下一交易日 |
| 总资产止损 | 发送紧急通知，**永久**停止交易 |
| 止损触发 | 立即止损卖出，发送通知 |

## 6. 数据来源与限制

| 数据项 | 来源 | 说明 |
|--------|------|------|
| 基准价 | AkShare/聚宽/其他 | 前一交易日收盘价 |
| 当前价格 | AkShare/聚宽/其他 | 可能有 15 分钟延迟 |
| ETF 代码 | config.yaml | 510300 沪深300ETF |

**聚宽试用限制**：
- 数据范围：距今 15 个月前 ~ 最近 3 个月
- 无实时行情权限（免费账号）

**AkShare 限制**：
- 免费，无需 API Key
- 偶有不稳定，建议配合其他数据源使用

**系统状态**：目前仅支持模拟交易，不支持实盘。

## 7. 常见问题

**Q1: 为什么选择沪深300ETF？**
- 波动性适中，适合网格交易
- 费率低、流动性好
- 无 T+1 限制（当天买的可以当天卖）

**Q2: 网格交易适合什么行情？**
- **最适合**：震荡行情（价格上下波动）
- **较适合**：小幅上涨行情（卖出的格子更高）
- **不适合**：单边下跌行情（持续买入不卖出）

**Q3: 10000元够做网格吗？**
- 每格约 500 元，最多持仓 5000 元（10格）
- 理论上够用，建议预留一些资金作为缓冲

**Q4: 趋势过滤在什么情况下会失效？**
- MA20 在市场盘整时频繁金叉/死叉，导致模式切换
- 建议配合 `confirm_days: 1`（连续1天确认）减少噪音

**Q5: 如何判断系统运行正常？**
- 查看 Streamlit 监控面板（:8501）的状态
- 微信收到每日总结通知表示当日运行正常
- 查看 `logs/` 目录下的日志文件
